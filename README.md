### Why MVVM

- MVP
    - Tight coupled 
        - Activity/Fragment need Presenter
        - Presenter hold ref to activity
        - Relationship is the issue
        - Quick Changes in UI need whole change
- MVVM
    - Google Introduced ViewModel
    - Google support MVVM
    - ViewModel has no idea about View
    - ViewModel easy to test - No view ref, No consumer behavior, No View dependency
    - ViewModel can be unit tested and NO Instrumentation needed
